# هندسة السياق (Context Engineering)

> المصدر: [addyosmani/agent-skills](https://github.com/addyosmani/agent-skills) — رخصة MIT

## المبدأ

أطعم الوكيل المعلومة الصح بالوقت الصح. السياق هو أكبر رافعة لجودة مخرجات الوكيل — قل جداً → هلوسة، كثر جداً → تشتت.

## هرم السياق

```
┌─────────────────────────────────────────┐
│  1️⃣ الذاكرة والمهارات (دائم)            │
├─────────────────────────────────────────┤
│  2️⃣ قواعد المشروع (USER.md, AGENTS.md)  │
├─────────────────────────────────────────┤
│  3️⃣ التوثيق المعماري (Specs, ADRs)      │
├─────────────────────────────────────────┤
│  4️⃣ الملفات المصدرية ذات العلاقة        │
├─────────────────────────────────────────┤
│  5️⃣ مخرجات الأخطاء ونتائج الاختبارات    │
├─────────────────────────────────────────┤
│  6️⃣ سجل المحادثة (يتراكم)              │
└─────────────────────────────────────────┘
```

## القواعد الذهبية

1. **لا تحمل ملفات لا تحتاجها** — كل ملف يحرق tokens
2. **استخدم البحث بدل التخمين** — `search_files` + `read_file`
3. **اضغط السياق المتكرر** — الحلقات التكرارية تأكل tokens بسرعة
4. **قسّم المهام الكبيرة** — استخدم subagents للمهام المتوازية
5. **اكتب الملخصات** — بعد كل مرحلة، لخص قبل الانتقال
