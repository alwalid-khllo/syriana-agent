# التطوير بالشك (Doubt-Driven Development)

> المصدر: [addyosmani/agent-skills](https://github.com/addyosmani/agent-skills) — رخصة MIT

## المبدأ

الجواب الواثق ليس بالضرورة جواباً صحيحاً. الجلسات الطويلة تراكم سياقاً يحوّل الافتراضات إلى "حقائق". التطوير بالشك هو تدقيق قبل أن يثبت أي قرار غير بديهي.

## العملية

### 1. الادعاء (Claim)
حدد القرار في سطرين + لماذا يهم.

### 2. الاستخراج (Extract)
أصغر وحدة قابلة للمراجعة — الـ diff أو الدالة، ليس الملف كامل.

### 3. الشك (Doubt)
"أين يمكن أن يخطئ هذا الادعاء تحديداً؟ ما الحالة الحدية غير المعالجة؟"

### 4. التوفيق (Reconcile)

| التصنيف | المعنى | الإجراء |
|---------|--------|---------|
| 🐛 خلل حقيقي | الادعاء خطأ | أصلح |
| ⚠️ حالة غير مغطاة | الادعاء صحيح لكن غير كامل | وسّع |
| 💡 تحسين | الادعاء صحيح والكود يعمل | طوّر |
| ❌ رفض | التشكيك لا ينطبق | تجاهل |

### 5. التوقف
توقف عندما: (1) كل النتائج "رفض" (2) 3 دورات شك (3) المستخدم قال كفاية.
