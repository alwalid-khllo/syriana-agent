# مراجعة الكود والجودة (Code Review & Quality)

> المصدر: [addyosmani/agent-skills](https://github.com/addyosmani/agent-skills) — رخصة MIT

## المبدأ

مراجعة كود متعددة الأبعاد قبل كل دمج. **اعتمد التغيير عندما يُحسّن صحة الكود العامة، حتى لو لم يكن مثالياً.**

## المراجعة الخماسية

### 1️⃣ التصحيح (Correctness)
- هل يطابق الـ spec؟ هل الحالات الحدية معالجة؟ هل مسارات الأخطاء مغطاة؟

### 2️⃣ القراءة والبساطة (Readability)
- أسماء وصفية؟ تسلسل تحكم مباشر؟ هل هناك كود ميت؟ هل التجريدات تستحق تعقيدها؟

### 3️⃣ البنية (Architecture)
- هل يتبع الأنماط الموجودة؟ هل يحافظ على حدود الوحدات نظيفة؟ هل هناك circular dependencies؟

### 4️⃣ الأمان (Security)
- مدخلات موثّقة؟ أسرار مخفية؟ استعلامات parameterized؟ مخرجات مشفّعة ضد XSS؟

### 5️⃣ الأداء (Performance)
- N+1 queries؟ تخزين مؤقت مناسب؟ موارد تُغلق بشكل صحيح؟

## قواعد
- لا استثناءات — كل تغيير يُراجع
- النقد للكود وليس للشخص
- اشرح "لماذا" — ليس فقط "هذا خطأ"
- قسّم المراجعات الكبيرة (400 سطر ← 4 مراجعات كل 100 سطر)
