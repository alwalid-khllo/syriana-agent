# المراقبة والقياس (Observability & Instrumentation)

> المصدر: [addyosmani/agent-skills](https://github.com/addyosmani/agent-skills) — رخصة MIT

## المبدأ

كود لا تستطيع مراقبته هو كود لا تستطيع تشغيله. القياس ليس إضافة بعد الإطلاق — يكتب مع الميزة.

## العملية

### 1. عرّف "الناجح" قبل القياس
اكتب 2-4 أسئلة سيطرحها مهندس on-call عن هذه الميزة قبل إضافة أي قياس.

### 2. اختر الإشارة المناسبة

| الإشارة | تجيب عن |
|---------|---------|
| **log منظم** | "ماذا حدث في هذه الحالة تحديداً؟" |
| **metric** | "كم مرة / بكم سرعة، بشكل إجمالي؟" |
| **trace** | "أين ذهب الوقت عبر الخدمات؟" |

### 3. التسجيل المنظم
- سجّل events وليس prose — كل سطر log هو JSON
- استخدم: `error` (خرق قاعدة), `warn` (متدهور), `info` (حدث تجاري), `debug` (تشخيص)
- **Correlation IDs إلزامية** — بدونها لا يمكن إعادة بناء طلب من سجلات متداخلة

### 4. التنبيه
- نبّه على الأعراض لا الأسباب
- نبّه على الاتجاهات لا القيم المنفردة
- كل تنبيه يؤدي لإجراء — وإلا احذفه
